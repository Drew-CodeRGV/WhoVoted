========================================
RUN THIS COMMAND ON THE SERVER
========================================

SSH into the server and run:

cd /var/www/politiquera && bash deploy/complete_llm_fix.sh

This will:
1. Pull latest code from GitHub
2. Clear Python cache
3. Verify Ollama is running
4. Restart gunicorn with proper timeouts
5. Run comprehensive tests
6. Verify HTTP endpoint
7. Show recent logs

The script will output detailed progress and results.

========================================
ALTERNATIVE: Step-by-step commands
========================================

If you prefer to run commands manually:

# 1. Pull code
cd /var/www/politiquera
git pull origin main

# 2. Clear cache
find backend -type d -name __pycache__ -exec rm -rf {} +
find backend -name "*.pyc" -delete

# 3. Restart gunicorn
pkill -9 -f "gunicorn.*app:app"
cd backend
source venv/bin/activate
nohup gunicorn -w 4 -b 127.0.0.1:5000 --timeout 120 app:app > logs/gunicorn.log 2>&1 &

# 4. Test
cd /var/www/politiquera
python3 deploy/test_user_queries.py

# 5. Monitor logs
tail -f backend/logs/error.log

========================================
WHAT WAS FIXED
========================================

The hanging issue was caused by Ollama calls blocking gunicorn workers
without any timeout. The fix:

1. Replaced signal-based timeouts (Unix-only) with threading.Timer
   (works on all platforms and in threaded environments)

2. Added 30-second timeout for SQL generation
   - If Ollama doesn't respond in 30s, returns timeout error
   - Prevents worker from hanging indefinitely

3. Added 15-second timeouts for explanations and suggestions
   - Non-critical operations that can fail gracefully
   - Falls back to simple text if timeout occurs

4. Enhanced logging to track where requests hang
   - "Calling Ollama with 30s timeout..."
   - "Ollama call completed"
   - Helps identify if issue is before/during/after Ollama call

5. Better error handling for TimeoutError exceptions
   - Returns user-friendly error messages
   - Doesn't crash the worker

========================================
TESTING THE FIX
========================================

After deployment, test in browser:

1. Go to https://politiquera.com
2. Sign in with Google
3. Click brain icon (🧠) in bottom-right
4. Enter: "Show me Female voters in TX-15 who voted in 2024 but not 2026"
5. Should see results within 30-45 seconds

If it still hangs:
- Check logs: tail -f /var/www/politiquera/backend/logs/error.log
- Look for "Calling Ollama with 30s timeout..."
- If you see that but not "Ollama call completed", Ollama itself is hanging
- Try: ollama list (to verify Ollama is responsive)
- May need to restart Ollama service
