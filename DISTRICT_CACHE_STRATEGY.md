## Current Implementation

### How Campaign District Cards Work

The system uses a **hybrid approach** with 3 levels:

#### Level 1: Pre-computed Cache (FASTEST - Instant)
- Location: `/opt/whovoted/public/cache/district_report_*.json`
- Generated by: `deploy/regenerate_district_cache_complete.py`
- When used: If cache file exists and is complete
- Speed: Instant (0ms) - just reads JSON file
- Status: **OUTDATED** - needs regeneration with new district assignments

#### Level 2: Database Query with District Columns (FAST - 25-200ms)
- Uses: `voters.congressional_district`, `voters.state_senate_district`, `voters.state_house_district`
- Query: `SELECT * FROM voters WHERE congressional_district = '15'`
- Speed: 25-200ms with indexes
- Status: **NOW AVAILABLE** - you just assigned all districts!

#### Level 3: Polygon Lookup (SLOW - 5-30 seconds)
- Uses: Point-in-polygon geometry calculations
- When: No cache, no district columns
- Speed: Very slow for large districts
- Status: Fallback only
